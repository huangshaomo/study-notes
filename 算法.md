# 数据结构与算法

> 数据结构：可以容纳数据的结构被称为数据结构，可以理解为一个箱子，用来装数据的，有各种各样的箱子，比如方形的，长条的，数据结构也同样如此，有链式的数据结构、二元的数据结构、线性的数据结构等等...

> 算法：算法是用来对数据解构进行处理的方法，也就是说算法就是用来操作数据结构的，可以将算法就可以看作是搬东西的过程，算法会根据箱子内装的物品的不同而采取不同的搬运方法，比如贵重物品的箱子需要小心翼翼的搬，装衣服的箱子可以快速搬。

> 因此，总的来说，数据结构是静态的，算法是动态的

### 1. 线性数据结构之(数组)

#### 简介

>  （Array）是一种线性表数据结构。它用一组连续的内存空间，来存储一组具有相同类型的数据。 



#### 数组特性

1. 存储在物理空间上是连续的

2. 底层的数组长度是不可变的（向操作系统请求分配一段内存空间）

3. 数组的变量，指向了数组第一个元素的位置

   

**优点**

查询性能好，指定查询某个位置

```javascript
var a = [1,2,3,4,5]
a[0],a[1],a[2] 其中方括号内数字表示存储地址的偏移量(offset)
```

![1580210800581](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230628.png)

**操作系统小知识**

通过偏移查询数据性能最好



**缺点：**

1. 因为空间必须是连续的，如果当数组长度比较大时，且系统的空间碎片较多的时候，操作系统难以在系统空间找到一个连续且比较大内存的空间，容易导致数组存不下，需要整理内存空间碎片到一起后才有可能腾出一块比较大的内存空间，因此非常消耗时间
2. 因为数组的长度是固定的，所以数组的内容难以 被添加和删除



#### 数组增删改查（图解）

如果数组要增加长度，首先需要向操作系统请求分配另一块更大的连续内存空间，然后将之前的数据复制过去，再将需要增加的数据排在最后一块

![数组长度增加](C:\Users\Administrator\Desktop\数组长度增加.jpg)

**插入**

 假设数组的长度为 n，将一个数据插入到数组中的第 k 个位置。为了把第 k 个 位置腾出来，给新来的数据，我们需要将第 k～n 这部分的元素都顺序地往后挪一位。 

![数组中插入数据](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230629.gif)

 如果在数组的末尾插入元素，那就不需要移动数据了，这时的时间复杂度为 O(1)。 

![数组末尾插入数据](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230630.gif)

但如果在数组的开头插入元素，那所有的数据都需要依次往后移动一位，所以最坏时间复杂度是 O(n)。 因为我们在 每个位置插入元素的概率是一样的，所以平均情况时间复杂度为 (1+2+…n)/n=O(n)。

如果数组中的数据是有序的，我们在某个位置插入一个新的元素时，就必须按照刚才的方法搬移 k 之后的数据。但是，如果数组中存储的数据并没有任何规律，数组只是被当作一个存储数据的集 合。在这种情况下，如果要将某个数组插入到第 k 个位置，为了避免大规模的数据搬移，我们还有 一个简单的办法就是，直接将第 k 位的数据搬移到数组元素的最后，把新的元素直接放入第 k 个位 置。

![快排思想](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230631.gif)



**删除**

插入数据类似，我们要删除第 k 个位置的数据，为了内存的连续性，也需要搬移数据，不然中间就会出现空洞，内存就不连续了。在某些特殊场景下，我们并不一定非得追求数组中数据的连续性。如果我们将多次删除操作集中在一起执行，这种就类似于Java虚拟中的标记清除。 

![标记清除](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230632.gif)

### 2. 线性数据结构之（链表）

 #### 什么是链表？

> 链表由一系列节点组成，每个节点一般至少会包含两部分信息：一部分是元素数据本身，另一部分是指向下一个元素地址的指针。这样的存储结构让链表相比其他线性的数据结构来说，操作会复杂一些。 

![1580554620837](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230633.png)

**用JS模拟一个链表**

```javascript
    this.next = null
```

#### 链表的特点

1. 空间上不是连续的
2. 每存放一个值，都要多开销一个引用空间

**优点**

1. 只要内存足够大，数据就能存的下，不用担心空间碎片的问题
2. 链表的添加和删除非常容易

**缺点**

1. 查询速度慢（指的查询某个位置）

2. 链表每一个节点都需要额外创建一个指向next的引用，浪费一些空间

   （不过当节点内数据越多的时候，这部分分销的内存影响越少）

```javascript
function getIndexNode(root,index){
    if(root == null) return;
    function _getValue(node,i){
        if(node == null) return null;
        if(i == index) return node;
        return _getValue(node.next,i+1)//这里也要return ,因为上面判定不过时这层函数最终也需要递归返回正确的值
    }
    return _getValue(root,0)
}

```

思路：先声明一个函数，参数1为根节点，参数2为目标下标索引，由于递归内部不能直接声明函数，会导致在递归时出现变量覆盖，因此不能直接声明一个索引值，而是再声明一个内部函数，参数1为节点，参数2为节点对应的下标，在内部函数内，如果定义的下标与目标下标索引一致，则直接返回该节点，否则继续递归该内部函数，传入下一个节点，并且定义的下标索引+1,如果最终都找不到，返回null

#### 常见面试题

##### 1. 链表的逆置

> 所谓逆置，就是将链表的指向反过来。链表逆置的 点就是先找到最后一个节点，因为最后一个节点的指向为空，这就是突破口  

**对于递归，必须先想出口**  

```javascript
function Node(value){
    this.value = value;
    this.next = null;
}
var node1 = new Node(1)
var node2 = new Node(2)
var node3 = new Node(3)
var node4 = new Node(4)
var node5 = new Node(5)

node1.next = node2;
node2.next = node3;
node3.next = node4;
node4.next = node5;

//逆置链表，参数为接受一个链表的根节点，出口为第五节点的next为空。
function reverseLink(root){
    if(root == null) return;
    if(root.next.next === null) {
        root.next.next = root;
        return root.next;
    }else{
        var result = reverseLink(root.next);
        root.next.next = root;
        root.next = null;
        return result;
    }
}
console.log(reverseLink(node1))
```

> 流程解析：
>
> 首先，调用该逆置函数，传入节点1，判断节点1的下家的next指向是否为空，不为空则继续递归调用逆置函数，传入节点2,以此类推...，直到递归调用逆置函数传入节点4时，发现节点4的下家（节点5）的next指向为空，将节点5的next指向节点4，此时传入节点4 的递归函数结束，并将节点5返回被result接受，此时的root回到了节点3的逆置函数环境中，将节点3的下家（节点4）的next指向节点3，再将节点3指向置空，然后节点3逆置函数执行结束，将修改后的节点5继续返回给节点2的逆置函数的result接受，然后将节点2的下家（节点3）的next指向节点2，将节点2 的next指向置空，然后节点2逆置函数结束，将修改后的节点5返回给节点1的逆置函数的result接受，继续将节点1的下家（节点2）的next指向节点1，将节点1指向置空，最终将修改完成的节点5链表返回。

##### 2. 快慢指针例子

它有一个很重要的性质：慢指针走的长度等于快慢指针相距的长度



##### 2.1. 倒数第K个节点

题目：**已知一个单链表的头节点，找到该链表中，倒数第K个节点**

 这个问题，主要的核心点在于，我们不知道单链表的长度，否则在遍历的时候，用一个 index 就可以解决。 

 如果我们想，只通过一次循环遍历，找到倒数第 K 个节点，就需要用到快慢指针了。 

![Rec 0018](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230641.gif)

设置两个指针，slow，fast，先让fast走k-1步，然后再一起走，直到fast为最后一个 时，slow指向的即为倒数第k个节点 

![1582432793769](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230642.png)

代码实现如下：

```javascript
function Node(value){
    this.value = value;
    this.next = null;
}
var node1 = new Node(1)
var node2 = new Node(2)
var node3 = new Node(3)
var node4 = new Node(4)
var node5 = new Node(5)
var node6 = new Node(6)
var node7 = new Node(7)
var node8 = new Node(8)

node1.next = node2;
node2.next = node3;
node3.next = node4;
node4.next = node5;
node5.next = node6;
node6.next = node7;
node7.next = node8;

function FindKthToTail(root,k){
    if(root == null) return;
    var slow = root;
    var fast = root;
    var cord = 1;
    function _FindKthToTail(slow,fast){
        if(fast.next == null){
            return slow;
        }
        else if(cord !== k){
            cord+=1;
			return _FindKthToTail(slow,fast.next)
        }else{
			return _FindKthToTail(slow.next,fast.next)
        }
    }
    return _FindKthToTail(slow,fast)
    
}
console.log(FindKthToTail(node1,3));

```

思路： 设置两个指针，slow，fast，（如果cord 小于k,先让fast走），直到fast走k-1步，然后再一起走，再直到fast为最后一个 时，slow指向的即为倒数第k个节点 



##### 2.2.找出单链表的中间节点

![找出单链表的中间节点](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230643.png)



思路如下：**我们把一个链表看成一个跑道，假设a的速度是b的两倍，那么当a跑完全程后，b刚好跑一半，以此来达到找到中间节点的目的。** 

```javascript
function Node(value){
    this.value = value;
    this.next = null;
}
var node1 = new Node(1)
var node2 = new Node(2)
var node3 = new Node(3)
var node4 = new Node(4)
var node5 = new Node(5)
var node6 = new Node(6)
var node7 = new Node(7)

node1.next = node2;
node2.next = node3;
node3.next = node4;
node4.next = node5;
node5.next = node6;
node6.next = node7;

//找出单链表的中间节点
function getMidNode(root){
    if(root == null || root.next.next == null) return;
    var slow = root;
    var fast = root;
    function _getMidNode(slow,fast){
        if(fast.next == null) return slow;	//奇数情况
        else if(fast.next.next == null) return [slow,slow.next]	//偶数情况
        else return _getMidNode(slow.next,fast.next.next)
    }
    return _getMidNode(slow,fast) 
}
console.log(getMidNode(node1));
```

思路：首先将快慢指针都指向起始节点，然后快慢指针开始移动，快指针的移动速度是慢指针的两倍(两个指针移动的位置都是相对于上次本身的位置进行移动, slow = slow.next  fast = fast.next.next)，临界点就是当快指针的next指向为空，指针不在移动，此时慢指针的位置就在中间。【单数链表得慢指针，偶数链表得慢指针+慢指针的next】





以上全部基于单向链表

### 2. 树形结构 --有向无环图

> 树形结构没有回路，类似文件夹间嵌套，树是属于拓扑图的一种,所谓无环，就是没有形成回路（不形成封闭的圈）

![树形结构](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230648.png)

#### 树的特点：

1. 单根（向）：如果一个节点A指向了另一个节点B，仅能通过A直接找到B节点，不可能通过其他节点直接找到B节点
2. 无环：节点的指向不能形成环

#### 树的术语：

1. 节点的度（叉）：某个节点的叉有几个（对于A节点来说，度为3,对于B节点来说，度为2）
2. 树的度：一棵树中，最大的（节点的度）为该树的度（明显是A,为3）
3. 节点的层：从根节点定义起，根为第一层，根的子节点为第二层，以此类推
4. 树的高度（深度）：树中节点的最大层次（B ）
5. 叶子节点：度为0的节点（就是没有叉的）
6. 分支节点：非叶子节点（有叉的节点）
7. 子节点、父节点 ：如果A是B的父节点，那么B就是A的子节点
8. 兄弟节点：如果两个节点有共同的父节点，则他们互为兄弟节点
9. 祖先节点：某个节点的祖先节点，是从树的根到该节点本身经过的所有节点（父亲也算祖先）
10. 后代节点：如果A是B的祖先节点，那么B则是A的后代节点（子节点也算后代节点）
11. 根节点：相对于某一棵树或某一个子树，树最顶端的节点，

（每一个节点都认为自己是根节点），每一颗节点都可以是一棵子树



#### 树的代码表示法：


#### 二叉树的遍历

前序遍历——先根次序遍历：**先打印当前的**，再打印左边的，再打印右边的

中序遍历——中根次序遍历：先打印左边的，**再打印当前的**，再打印右边的

后序遍历——后根次序遍历：先打印左边的，再打印右边的，**再打印当前的**

> 可以看出，前中后是针对当前

##### 前序遍历

```ACFGBDE``` 对于前序遍历，先当前、再左边。后右边，对于A来说，A的前序遍历当前就是自己，所有先打印(A),然后A的左边是子树C,对于子树C的前序遍历来说，C是当前，先打印当前(C)，然后打印左边（F）,再打印右边（G）,然后，子树C的前序遍历结束，回到A的前序遍历中，A的当前和左边打印完后，来到A的右边子树B中，对于子树B的前序遍历来说，先打印当前，也就是（B）,再打印左边（D）,最后打印右(E)。

#### 图的最小生成树问题

> 图与树最大的不一样是树只有父节点指向子节点，而图是来回指向（A指向B，B指向A）

>有两种算法： 一种是普利姆算法，一种是克鲁斯卡尔算法

##### 普利姆算法（加点法）

1. 任选一个点作为起点
2. 找到以当前选中点为起点路径最短的边
3. 如果这个边的另一端没有被连通进来，那么可以连接
4. 如果这个边的另一端也早被连进来，则看倒数第二短的边
5. 重复2-4直到所有的点都被连通为止

```javascript
function getMinDisNode(pointSet,distance,nowPointSet){
    var fromDisNode = null;	//线段的起点
    var endDisNode = null;	//线段的终点
    var minDis = max;		//默认最短距离
    //根据当前已有点为起点，依次判断连接其他的点的距离是多少
    for(var i = 0; i < nowPointSet.length; i++){
        var nowPointIndex = getIndex(nowPointSet[i].value)	//获取当前节点在点集合中的索引
        for(var j = 0; j < distance[nowPointIndex].length; j++){
            var thisNode = pointSet[j];		//获取边在数组里对应的另一个点，因为有一一对应的
            //如果当前节点是未接入的点并且点与点之间的距离最小，则满足普利姆算法条件
            if(nowPointSet.indexOf(thisNode) < 0 && distance[nowPointIndex][j] < minDis){
                fromDisNode = nowPointSet[i];
                endDisNode = thisNode;
                minDis = distance[nowPointIndex][j];
            }
        }
    }
    //找到最小距离的点后，需要将两个点的关系连起来，互相指向对方
    fromDisNode.neighbor.push(endDisNode);
    endDisNode.neighbor.push(fromDisNode);
    //最终，被连结的点需要放到连结数组（nowPointSet）中，表示该点已被连结。
    return endDisNode;
}
```

**实现思路**：

首先定义一个接受三个参数的函数，分别把点的集合、边的集合、起始点当做参数传递进去，然后建立一个存放已被连结的点的数组，开始时默认只有起始点一个，然后找到该点在点集合中的下标索引，代入到边索引中（因为边索引是依据点索引建立的，因此下标对应的点是一样的），找到与该点关联的数组，然后遍历该数组，将该数组的没个值逐个与最小值比较，每遍历一遍，把比最小值还小的值赋给最小值，直到循环结束，会得到当前点的最小边对应的点，该点就是最小边对应的点，找到该点后，将两个点的关系连起来，将对方的neighbor互相指向对方,最后将该点返回回去给push到存放已连结的点的数组中（nowPointSet），最后，如果该数组的长度等于点的集合的长度，说明所有最短边对应的节点已经全部被被连结上，则退出循环





##### 克鲁斯卡尔算法（加边法）

1. 选择的边进行连结

2. 要保证边连结的两端至少有一个点是新的点

   或者这连个边是将两个部落进行连结的

3. 重复1-2直到将所有的点连结到一起

```javascript
function canLink(resultList, tempBegin, tempEnd) {
    //首先查看两个节点是否有所属部落
    var beginIn = null;
    var endIn = null;
    for (var i = 0; i < resultList.length; i++) {
        if (resultList[i].indexOf(tempBegin) > -1) {
            beginIn = resultList[i];
        }
        if (resultList[i].indexOf(tempEnd) > -1) {
            endIn = resultList[i];
        }
    }
    // 两个点都是新的点（都不在任何部落里） ——可以连接
    // begin在A部落，end没有部落 ——A部落扩张一个村庄
    // begin没有部落，end在B部落 ——B部落扩张一个村庄
    // begin在A部落，end在B部落 ——A,B部落合并
    // begin和end都在同一个部落 ——不可以连接
    if (beginIn != null && endIn != null && beginIn == endIn) {
        return false
    }
    return true;
}

function link(resultList, tempBegin, tempEnd) {
    var beginIn = null;
    var endIn = null;
    for (var i = 0; i < resultList.length; i++) {
        if (resultList[i].indexOf(tempBegin) > -1) {
            beginIn = resultList[i];
        }
        if (resultList[i].indexOf(tempEnd) > -1) {
            endIn = resultList[i];
        }
    }
    // 两个点都是新的点（都不在任何部落里） ——两个点组成一个部落再放入到部落列表中
    if (beginIn == null && endIn == null) {
        var newArr = [];
        newArr.push(tempBegin);
        newArr.push(tempEnd);
        resultList.push(newArr);
    } else if (beginIn != null && endIn == null) { // begin在A部落，end没有部落 ——A部落扩张一个村庄
        beginIn.push(tempEnd);
    } else if (beginIn == null && endIn != null) {// begin没有部落，end在B部落 ——B部落扩张一个村庄
        endIn.push(tempBegin);
    } else if (beginIn != null && endIn != null && beginIn != endIn) {// begin在A部落，end在B部落 ——A,B部落合并
        var allIn = beginIn.concat(endIn);
        var needRemove = resultList.indexOf(endIn);//获取该部落在部落列表中的索引
        resultList.splice(needRemove, 1); //将部落B从部落列表中移除
        needRemove = resultList.indexOf(beginIn)//获取该部落在部落列表中的索引
        resultList.splice(needRemove, 1); //将部落A从部落列表中移除
        resultList.push(allIn);	//合并后的A部落加入部落列表
    }
    //建立关系
    tempBegin.neighbor.push(tempEnd);
    tempEnd.neighbor.push(tempBegin);

}

function kruskal(pointSet, distance) {
    var resultList = []; //这是一个二维数组，数组代表的是有多少个部落。
    while (true) {
        var minDis = max;
        var begin = null;
        var end = null;
        for (var i = 0; i < distance.length; i++) {
            for (var j = 0; j < distance[i].length; j++) {
                var tempBegin = pointSet[i]; //获得起始边
                var tempEnd = pointSet[j]; // 获得结束边
                //console.log(pointSet[i],pointSet[j],distance[i][j]);
                //去掉自己到自己的距离，去掉大于最小值得距离，去掉验证不通过的
                if (i != j && distance[i][j] < minDis && canLink(resultList, tempBegin, tempEnd)) {
                    //如果满足以上条件，说明是最小的边了，得到最小代价的两个点跟一个边。
                    minDis = distance[i][j];
                    begin = tempBegin;
                    end = tempEnd;
                }
            }
        }
        //建立归属问题
        link(resultList, begin, end);
        //如果最终部落只有一个且部落人数的个数等于点的集合的个数，则结束循环
        if (resultList.length == 1 && resultList[0].length == pointSet.length) {
            break;
        }
    }

}
```

- 不能为自己到自己的距离，因为都是0

- 必须是代价为最小的边

- 必须通过下面的验证(canlink)

  - 一个有部落，一个没有部落

#### 二叉树的单旋操作（左单旋，右单旋）

> 如果左边浅，右边深，进行左单旋（右边的向左旋转）

#### 二叉树的双旋（左右双旋，右左双旋）

概念：为什么需要二叉树的双旋，因为单旋在某些场景下无法将二叉树变为平衡二叉树

![二叉树的双旋](https://hsm-typora-img.oss-cn-beijing.aliyuncs.com/img/20210105230711.png)



**需要进行双旋的情况：**

当要对某个节点进行左单旋时，

**如果变化分支变为是唯一的最深分支**，那么我们要对新根进行右单旋然后再进行左单旋

这样的旋转叫做右左双旋。这样的目的是为了削减变化分支的长度



当要对某个节点进行右单旋时

**如果变化分支变为是唯一的最深分支**，那么我们要对新根进行左单旋然后再进行右单旋

这样的旋转叫左右双旋。这样的目的是为了削减变化分支的长度

### 4. 234树

  我们希望有一棵树，最多有4个叉（度为4）

我们会发现一个特点，新来的节点永远都在最下层。由于新来的永远在最下层，因此这棵树永远都是平衡的（每一个路径高度都相同）

优点

- 分支变多了，层数边少了
- 节点中存的树变多了，节点变少 了

缺点

- 因为分支多了，所以复杂度上升了



234树又引发了另一个问题：因为分支多了，所以复杂度上升了，那我们该如何对234树进行简化呢？

- 希望能简化为二叉树
- 希望依旧保留多叉
- 希望依旧单节点中能存放多个值

而这些无理而矛盾的要求，让人推导出了红黑树。



### 5. 红黑树

#### 动态规划之青蛙跳台阶

问题：一个青蛙，一次只能跳一级台阶，或者跳两级台阶，问：这个青蛙跳上n级台阶有多少种跳法。

解析：如果这只青蛙，跳上了第n级台阶，那么在进行**最后一次**跳跃之前，他一定在n-1级台阶或n-2级台阶，（这里需要注意的是，最进行最后一次的跳跃）那么跳上n级别台阶有多少种情况就变成了两个子问题。跳上n-1级台阶的跳法+跳上n-2级台阶的跳法。继续按照此逻辑递推，跳上n-1级台阶可以拆解为两个子问题，即：跳上n-2级台阶的跳法 + 跳上n-3级台阶的跳法。

因此，我们得出一个公式:`f(n) = f(n-1) + f(n-2)`，可以发现好像是菲波拉契数列。



```javascript
function jump(n){
    if(n <= 0 ) return -1;
    if(n == 1) return 1;
    if(n == 2) return 2;
    return jump(n-1)+jump(n-2);
}
jump(4)
```

```javascript
    var existed = [];
    function _jump(n){
        var newRecord
        //查看序列对应的数值存不存在，存在直接返回该数值
        if(existed[n]) return existed[n];
        //如果序列对应的数据不存在，说明还未添加进去，继续判断n
        if(n <= 0)  newRecord = -1
        else if(n === 1) newRecord = 1
        else if(n === 2) newRecord = 2
        else{
            newRecord =  _jump(n-1) + _jump(n-2);
        }
		existed[n] = newRecord
        return newRecord
    }
    console.log(existed);
    return  _jump(n)
}
```

思路：首先定义一个函数，参数1为几级台阶，返回跳上该级台阶一共有几种方法，然后定义一个函数，为了该函数全局可用，在内部再定义一个函数。首先，先判断数组里是否存在下标为n的数据，如果有，说明当前数据已经保存在数组里，直接返回对应下标的数据，如果没有该下标，声明一个变量用来记录值，如果n是一些特殊次，将特殊位的值赋给newRecord，然后将当前n对应给数组的第n位添加值，再把值返回出去给上级，下次再找n位的数值的时候，可以直接从existed中拿到， 如果n不是特殊次，进入递归，将最终得到的值赋给newRecord，然后将当前n对应给数组的第n位添加值，这样非特殊次也存在值在数组中的，然后再把值返回出去给上级

## 排序算法

> 排序本质上不是比较大小，而是比较和交换.
>
> 任何一种排序算法，都没有优劣之分，只有是否合适的场景
>
> 越混乱越适合快速排序，越有序越适合冒泡排序

原地（就地）排序：不开辟新的内存空间

下面所有的算法，都是原地排序



### 冒泡排序

一次冒泡排序，可以将某个区域序列的最大值排序到该区域的最后一位，具体方式是：

1. 将第1位和第2位比较，如果前者比后者大则交换
2. 将第2位和第2位比较，如果前者比后者大则交换
3. 以此类推，直到比较到该区域的最后两位
4. 重复上述过程，直到序列排序完成 

```javascript
var arr = [4,1,6,9,3,2,8,7]
function compare(a,b){
   if(a>b)return true;//a>b表示升序，因为a的位置在b的前面，a的值大于b的值，将a,b值互换的化就是小的放前面，就是升序了
    return false;
}
function exchange(arr,a,b){
    var temp = arr[a];
    arr[a] = arr[b];
    arr[b] = temp;
}

function bubbleSort(arr){
    if(arr == null || (arr != null && arr.length <= 1) ) return;
    for(var i = 0; i < arr.length-1; i++){//外层表示进行几次冒泡排序
        for(var j = 0; j < arr.length-i-1; j++){//内层表示比较的次数
            if(compare(arr[j],arr[j+1])){
               exchange(arr,j,j+1)
              
        }
    }
    return arr;
}
bubbleSort(arr)
```

> 规律：
>
> 外层循环：8个数，进行（8-1）次冒泡排序即可
>
> 内层循环：8个数，进行（8-1- i）次比较即可，因为一次比较需要两个数，每次冒泡排序后需要比较的数又少一个





### 选择排序

一个选择排序，可以将某个区间的最小值排列到该区域的第一位，具体的方式是：

1. 找出该区域的最小值
2. 将该值与该区域的第一个值交换
3. 对下一个区域重复上诉过程，直到排序完成

```javascript
var arr = [4,1,6,9,3,2,8,7]

    if(a > b) return true;//a>b升序   a<b降序

function exchange(arr,a,b){
    var temp = arr[a];
    arr[a] = arr[b];
    arr[b] =  temp;

		var minIndex = i;

                 minIndex = j
            }

    }
    return arr;

```

>
> 外层循环：8个数，进行（8-1）次选择排序即可，因为最后一次是自身与自身的比较，没有意义
>
> 内层循环：8个数，进行（i+1）次比较即可，1表示开头时不与自身比较，没意义，i表示比较过的不用在比较，因为比较过的放在了最前面。而不是像冒泡排序一样比较过的放在最后面。

### 插入排序 

将序列分为两个部分，一部分是有序的，一部分是无序的，现在要做的是，就是不断的从无序的部分取出数据，加入到有序的部分，直到整个排序完成

适合场景：差不多排完序的那种

例如：序列 [5，7，2，3，6]

1. 分为有序的序列和无序的序列（5）（7 2 3 6）
2. 不断地扩充有序序列（5 7）（2 3 6）
3. 不断地扩充有序序列（2 5 7）（3 6）
4. 不断地扩充有序序列（2 3 5 7）（6）
5. 不断地扩充有序序列（2 3 5 6 7 ）
6. 排序完成

```javascript
var arr = [5,3,1,4,7,6]

           var temp = arr[i]

               if(j> 0 && arr[j-1] > temp){
                   arr[j] = arr[j-1];//将arr[j-1]的值赋给arr[j],相当于后移
               }else{
                   arr[j] = temp;//将temp的值赋给arr[j]

               }
           }
          
        }

    }
    return arr
}
```

### 二分查找

如果一个序列是一个排序好的序列，则使用二分查找可以极大的缩短查找时间

具体的做法是：

1. 相等，找到
2. 找到的数据较大，则对后续部分的数据做同样的步骤
3. 要找的数据较小，则对前面部分的数据做同样的步骤

就是类似于二叉树搜索树，每次都与中间的值比较，每查询一次排除一半的数据量，

```javascript
function binarySearch(arr,target){
    if(arr.length === 0 || target < arr[0] || target > arr[arr.length-1]) return false
    var minIndex = 0;
    var maxIndex = arr.length -1
    while(minIndex <= maxIndex){
        var mid = Math.floor((minIndex+maxIndex)/2)//中间下标
        if(arr[mid] === target){
            return true
        }else if(arr[mid] > target){
            maxIndex = mid-1	//maxIndex减半
        }else{
            minIndex = mid +1	//minIndex减半
        }
    }
    return false
}
```



### 插值查找（类似新华字典的查找）

插值查找是对二分查找的进一步改进

弱序列不仅是一个排序好的序列，而且序列的步长大致相同，使用插值查找hi更快的找到目标

插值查找基于如下假设：下标之间的距离比和数据之间的距离比大致相同

即：

（目标下标-最小下标）/(最带下标 -最小下标) =（目标值 - 最小值）/(最大值 - 最小值)

因此可以算出大致的下标落点：

目标下标 = （目标值 - 最小值）/ (最大值 - 最小值)*（最大下标 - 最小下标）+最小下标

这样就可以计算出大致的下标落点，后续的比较和二分查找一样

```javascript
function interpolationSearch(arr,target){
    if(arr.length === 0 || target < arr[0] || target > arr[arr.length-1]) return false
    var minIndex = 0;
    var maxIndex = arr.length -1
    while(minIndex <= maxIndex){
        var mid = (target-arr[minIndex])/(arr[maxIndex]-arr[minIndex]) *(maxIndex-minIndex) + minIndex//中间下标
        if(arr[mid] === target){
            return true
        }else if(arr[mid] > target){
            maxIndex = mid-1	//maxIndex减半
        }else{
            minIndex = mid +1	//minIndex减半
        }
    }
    return false
}
interpolationSearch(arr,target)
```



##　贪心算法

当遇到一个求解全局最优解问题时，如果可以将全局问题切成分为小的局部问题，并**寻求局部最优解**，同时可以证明**局部最优解累计的结果就是全局最优解**，则可以使用贪心算法。

面试题：找零问题

示例：假设你有一间小店，需要找给客户４６分钱的硬币，你的货柜里只有面额为２５分、１０分、５分、１分的硬币，如何找零才能保证数额正确并且硬币数最小

```javascript
/**
*  贪心算法：找零问题
*  @params  total   需要找零的总额度
*  @params  deno	有哪些面额
**/
function exchange(total,deno){
    var result = []
    while(total > 0){
        var max = -1;
        for(var i = 0; i < deno.length; i++){
            //找到当前数组比total小的最大的面额
            if(deno[i] <= total&& deno[i] > max ){
                max = deno[i]
            }
        }
        result.push(max);
        total = total-max;
        console.log(max)
    }
    return result
}
console.log(exchange(46,[25,10,5,1]) )//返回最优解的数组
console.log(exchange(41,[25,20,10,5,1]))//返回最优解的数组，最优解是[20,20,1]，但返回的是[25,10,5,1]
```

贪心算法是非常简单的，每次只要找到当前小于额度的最大面额即可,但是，贪心算法没有回溯的过程，不会反思这样做到底对不对，由于只管眼前，没有全局思维，因此会导致找出的最优解不一定是真正的最优解



因此，贪心算法比较适合的场景是，不在乎一定是不是最优解的场景，反正都是比较好的，或者是你知道给出这样的数这样做出来的就是最优解，那就可以用这种算法
