#  window批处理

## 1. echo，rem，cd，dir

1. 显示信息：echo
2. 注释语句：rem
3. 目录切换：cd
4. 列文件名：dir

**echo**

**1. 输出提示信息**

> 格式：echo 你要输出的信息。

```cmd
//mybat.bat
echo hello world
```

执行显示

```
C:\bat>echo hello,world
hello,world
```

 我们写的批处理，如果不想要它显示执行命令语句（回显）的话，有两种方式。

1. 在echo语句前面加上@表示关闭这行语句的回显

   ```cmd
   @echo hello world
   hello bat
   ```

   执行显示

   ```
   hello world
   C:\bat>echo hello bat
   hello bat
   ```

   

2. 在顶部加上一句 @echo off，表示关闭回显指令。

   ```cmd
   @echo off
   echo hello world
   echo hello bat
   ```

   执行显示

   ```
   hello world
   hello bat
   ```

   

**2. 输出回车**

```cmd
@echo/
@echo. 
@echo,
@echo:
@echo;
```

上面这些都能输出回车



**2. 用于回答命令中的提问**

> 格式：echo 回复内容|提问内容(执行语句)
>
> 作用：通过管道命令 | 把“回复”作为输入传导给后面的“命令表达式”，并作为“命令表达式”的输入。（注
> 意，管道命令|的输入方法，shift键 \）

```cmd
@echo 10:10:10.10 | time
```

上面表示请求window读取当前时间，这时window会返回一个当前时间和一个输入新时间的内容框，`10:10:10.10`则会作为回复内容输入进去。

```
c:bat>echo 10:10:10.10|time
当前时间: 15:29:04.23
输入新时间: 10:10:10.10
```

2.

```cmd
rd /s c:\abc
```

上面表示删除(rd)C盘下的abc文件，/s表示除目录本身外，还将删除指定目录下的所有子目录和文件，此时操作window会询问你是否确认删除，填Y或N

```
c:\abc, 是否确认(Y/N)?
```

如果想要系统帮我确认YES的话，可以在echo  后面回复Y，window会把这个语句当做回复命令

```cmd
echo Y|rd /s c:\abc
```



**3. 建立新文件或添加文件内容**

> 格式：echo 文件内容  > 文件名

```cmd
@echo 123 > myfile.txt
```

上面表示给myfile.js文件内容替换为123，注意是替换，不是追加到最后，如果不存在该文件，则新建再加入。

新建一个空文本文件

```cmd

```

使用回车符号都能新建一个空白文件，否则就会存放一些默认内容，比如`echo处于打开状态`。

**3. 目录切换**

**1. 同盘符切换**

格式：cd [盘符]\[路径]

例子：C:\WINDOWS\system32

```
//上一级
cd..						相对路径方式
cd c:\WINDOWS				 绝对路径方式

//下一级
cd system32					 相对路径方式
cd c:\WINDOWS\system32		  绝对路径方式

//当前级
cd.							相对路径方式

//返回根目录
cd\							相对路径方式
cd c:\						绝对路径方式
```

**2.  不同盘符切换**

格式：cd  /d [盘符]\[路径]

例子：C:\WINDOWS\system32

```
//切换到E盘指定目录
cd /d e:\abc

//直接切换到E盘根目录
e:	
```

1. 注意\d是一种格式，而不是盘符名称
2. `e:`这种只能切换到根目录，后面不能再加路径了

**4. dir 查看目录**

> 结构： DIR\[drive:]\[path]\[filename] [/A[[:]attributes]] [/B] [/C] [/D] [/L] [/N]\[/O[[:]sortorder]] [/P] [/Q] [/S] [/T[[:]timefield]] [/W] [/X] [/4]
>
> 解释：
>
> \[drive:]\[path]\[filename]：指定要列出的盘符、目录和文件。
>
> A：显示具有指定属性的文件。【筛选功能】
>
> 后接`:attrbutes`表示筛选指定类型：D目录，R只读文件，H隐藏文件，A准备存档的文件，S系统文件，-表示“否”的前缀。
>
> B：使用空格式（没有标题信息或摘要，只显示文件或目录名称）
>
> C：在文件大小中显示千位数分隔符`,`例如(1,100)，这是默认值，用 /-C 来停用分隔符显示。
>
> D：以列表的格式列出文件名
>
> L：文件名全部转为小写
>
> O：用分类的方式列出文件
>
> 后接`:sortorder`表示分类方式：N按字母(正序)，S按从小到大，E按扩展名字母(正序)，D按日期/时间（从小到大），G文件夹优先展示，前面加`-`表示颠倒顺序
>
> P：只显示屏幕装得下的内容，并提供提示`按任意键继续`继续往后查看
>
> Q：显示文件拥有者
>
> S：显示指定目录和所有子目录中的文件。
>
> ...
>
> 以上的可忽略大小写。

常用的有A，B，P，O

**A**（筛选文件）

```
dir /a c:\		列出C盘下的所有文件
dir /ah c:\		列出C盘下的所有隐藏文件
dir /ad c:\		列出C盘下的所有文件夹
dir /ad-h c:\	列出C盘下的所有隐藏的文件夹
```

**B**（只显示文件名称）

```
dir /b c:\		只显示文件名称
```

```

**1. attrib（可操作文件）**

> 格式：ATTRIB [ R | -R] [ A | -A ] [ S | -S] [ H | -H] [[drive:] [path] filename] [/S [/D]]
>
> 解释：查看或更改文件属性。（不加条件是查看，加了条件是更改）
>
> 条件：
>
> —	清除属性
>
> +R：只读文件属性（read）
>
> +A：存档文件属性（archive）
>
> +S：系统文件属性（system）
>
> +H：因此文件属性（Hidden）
>
> [dirve:]\[path][filename]：指定要处理的文件路径
>
> /S：处理当前文件夹及所有子文件夹匹配的文件
>
> /D：处理

**1. 查看文件属性**

```cmd
attrib d:\abc.txt
```

查看D盘下的abc文件的属性，如果是

**2.修改文件属性**

```
attrib -h d:\abc.txt	//消除abc.txt文件的隐藏属性
attrib +s h	d:\abc.txt	//给abc.txt文件同时设置为系统文件并隐藏起来

**2. del（可操作文件，文件夹）**

> 两种格式：del   删除方式   删除文件路径。
>
> ​				 erase 删除方式  删除文件路径。
>
> ```
> DEL [/P] [/F] [/S] [/Q] [/A[[:]attributes]] names  或
> ERASE [/P] [/F] [/S] [/Q] [/A[[:]attributes]] names	  		
> ```
>
> 功能：用于删除文件或文件夹
>
> 删除文件路径：
>
> names:指定删除一个或数个文件或目录列表。使用通配符（*）可删除多个文件，如果删除某个目录，目录下的所有文件也会被删除。
>
> 删除方式：
>
> /P：删除每一个文件之前提升确认。（prompt）
>
> /F：强制删除只读文件。（force）
>
> /S：从所有子目录删除指定文件。（search），注意，不会删除文件夹
>
> /Q：安静模式。删除全局通配符时，不要求确认。(quiet)
>
> /A：根据属性选择要删除的文件。（attribute）R：只读文件，S：系统文件，H：隐藏文件，A：存档文件，-表示“否的前缀”。

1. 删除文件

```shell
del d:\123\abc.txt				//删除123文件夹下的abc.txt文件
del d:\123\*.txt				//删除123文件夹下的所有txt文件
```

2. 删除每一个文件之前提示确认(/P)

```
del/p d:\123\*.*					//删除123文件夹下的所有文件，并提示确认
```

3. 搜索目录下指定文件进行删除，搜索包括子目录(/S)

```
del /s d:\123\miui.txt				//删除123文件夹及子文件夹下的miui.txt
```

4. 使用全局通配符进行删除时，会提示确认删除，如果不想要这个确认，可以在搜索删除时加/Q

```
del /s d:\123\*.*					//这样写会提示是否确认删除
del /s /q d:\123\*.*				//这样就不会提示了
```

 	5. 根据属性选择需要删除的文件

```
del /ar /s d:\123\*.*			//搜索删除123目录及子目录下的所有只读属性的文件。
```



总结：

1. 删除文件夹或使用全局通配符进行删除时，就是提示是否确认对话框。





**3. copy（可操作文件，文件夹）**

> 格式：copy  源文件  目的文件
>
> ```
> copy source[drive:][path][filename]  [destination [drive:][path][filename]]
> ```
>
> 功能：复制文件

1. 目的文件不写filename，表示放到文件夹下

```
copy c:\123.txt  d:\			//将123.txt文件复制到d盘根目录下，文件名为源文文件名
```

2. 目的文件名与源文件名不一致，表示先把源文件复制到目的文件位置后，再把名称该位目的文件名

```
copy c:\123.txt d:\abc.bat		//把123.txt复制到d:\目录下，并把它修改为abc.bat。
```

3. 源文件为文件夹名，表示复制该文件夹里的文件到源文件中。

```
路径：c:\bat\abc
c:\bat > copy abc				//表示把当前文件夹(bat)下的abc文件夹中的文件复制到当前文件夹(bat)中
```



**批量复制文件**

> 当复制的源文件是文件夹时，就会变成批量复制，此时会把源文件下的所有子文件（不包括子文件夹和其内部的文件）复制到目标路径中。如果目标文件存在同名文件，会提示是否覆盖

```
copy c:\123 d\abc			将c:\123 文件夹中所有文件复制到d:\abc文件夹中.
```

```
copy c:\123\*.rm d:\abc /y		//将c:\123 目录中所有rm格式的文件复制到d:\abc目录中
```



**合并文件**

> 格式：copy /b 文件1 + 文件2 + ...文件N
>
> 使用 + 表示同时复制左右两边文件内容，/b表示使用二进制（binary）,例如像图片、歌曲，/a表示文本（ASCII），例如纯文本。合并不同类型的文件内容时，注意文件的数据是二进制还是文本，否则会导致合并失败。而且文本只能与文本合并，二进制只能与二进制文件合并，不然也会失败
>
> 不指定目标文件名的情况下，默认是第一个文件名的名称

1. MP3文件在使用此方法连接后，能实现连续播放。

```
copy /b d:\1.MP3 + d:\2.MP3 e:\3.MP3	//把1.MP3和2.MP3合并成3.MP3
```



2. 没有指定合成后的文件名

```
copy /b d:\1.txt d:\2.mp3			//把1.txt和2.mp3合并,单没有指定合成后的文件名，缺省情况下，合并后的文件名是命令中的第一个文件的名。
```



注意:

1. 在尾部隐藏了文本数据的图片文件，在使用其他软件进行编辑并保存后，隐藏的文本数据有可能会丢失。
2. MP3文件在使用此方法连接后，能实现连续播放。
3. 合并图片/歌曲这样的二进制文件必须使用/b参数(b代表Binary，二进制)，否则合并将会失败;另一个合并参是/a
   (a代表ASCII，文本文件)，只能用于纯文本的合并。两参数不能同时使用，二进制方式可以合并文本和二进制文件，
   而文本方式则只能合并文本。

## 3. md，rd，ren，move

1. 创建文件夹：md
2. 删除文件夹：rd
3. 重命名文件（夹）：rem
4. 移动文件（夹）

**md（可操作文件夹）**

> 格式：md [路径\]目录
>
> 基础写法：`md d:\abc`在D盘下建立一个名为abc的文件夹。
>
> 功能：md的命令只能用于创建文件夹，如果要创建文件，请使用 echo. > abc.txt

1. 不填写路径，默认只会在当前目录下创建文件夹

```
md abc		//这里缺省路径，则在当前目录下创建一个名为abc的文件夹。
```

2. 文件夹名包含有空格或是特殊符号，需要用"双引号"把整个文件夹名括起来(注意只能是双引号)，空格会被认为创建多个文件。

```
md "d:\my game"		//对于新建的文件夹名称中包含有空格或是特殊符号的，要用双引号” “把文件夹名括起来。
```

3. 创建多级目录

```
md d:\abc\abcd\abcde
```



**rd（可操作文件夹）**

> 格式：RMDIR [/S] [/Q] [drive:]path	或 	RD [/S] [/Q] [drive:]path
>
> 基础写法：`rd d:\abc`
>
> 功能：rd（renamedir）表示删除文件夹，如果要删除文件，请使用`del c:\abc.txt`。
>
> 指令：
>
> /s	删除当前文件夹，包括内部的所有子文件夹和文件，相当于删除当前文件夹的整个目录树
>
> /q	安静模式，带/s删除目录树时不要求确认

1. 默认情况，rd只能删除空目录，否则系统会提示目录不是空的，且删除失败

   ```
   rd abc		//删除当前目录下的abc文件夹，如果abc不为空则删除失败
   ```

2. /s

   ```
   rd /s abc	//删除abc整个文件夹，相当于右键删除，不过有删除提示
   ```

3. /q 安静模式，带 /s 删除目录树时不要求确认。

   ```
   rd /s/q abc	//等同于我们右键删除
   ```



**3. ren（可操作文件，文件夹）**

> 格式：rename \[drive:][path]filename1     filename2    或 REN\[drive:][path]filename1     filename2
>
> 基础写法：ren d:\abc\123.txt       456.bat
>
> 功能：ren表示重命名，可以给文件或文件夹进行命名，filename1的路径可以省略，默认情况下为当前目录，filename2只能是文件名，不能使用任何路径。
>
> 注意点：
>
> 1. 当文件具有隐藏属性或系统属性的时候，ren不能直接重命名，需要把文件的隐藏或系统属性去掉方可更名

1. 使用通配符可批量对文件命名

   ```
   ren *.bat *.txt			//将.bat后缀的文件都替换成.txt文件
   ```



**4. move（可操作文件，文件夹）**

> 格式：文件移动：move [磁盘]\[路径]\[文件名]\[文件名...]   目标文件夹路径
>
> 基础写法：`move d:\abc  d:\abcd`把文件夹abc移到文件夹abcd内。
>
> 功能：可以对文件或文件夹进行移动，如果目标文件夹不存在，会把源文件夹重命名成目标文件夹，并放到目标文件夹的位置。【根据特点，move也具备重命名的功能】
>
> 注意点：

1. A，B为文件

   1. 同一磁盘，如果B存在

   ```
   move a.txt b.txt	//提示：改写 D:\456.txt 吗? (Yes/No/All):，如果填Y，就会把B删除，然后把A变成B。
   ```

   2. 同一磁盘，如果B不存在

   ```
   move a.txt b.txt	//直接把A变成B
   ```



2. A，B为文件夹

   B存在，同分区

   ```
   move a b		//覆盖b并重命名a为b
   ```

   B存在，不同分区

   ```
   move c:\a  d:\b		//失败，系统提示拒绝访问
   ```

   

3. A为文件夹，B为文件

   如果B存在

   ```
   move A B.txt	//把A移动到B所在文件夹下，然后把B名字给A
   ```

   如果B不存在

   ```
   move A B.txt	//把A移动到B所在文件夹下，然后把名字改完B.txt
   ```

   

总结：

1. 无论是文件还是文件夹，目标地址存在就覆盖并重命名，不存在就移动并重命名
2. 如果操作的都是文件夹，如果两者都存在且在不同磁盘的话，无法成功，提示拒绝访问。（MOVE命令可以跨分区移动文件，却不可以跨分区移动文件夹。）
3. 对于跨分区移动文件时，如果目标位置存在和将要移动的文件同名时，系统会提示：“文件存在” 最后，命令将
   不会执行。
4. 对于隐藏或系统属性的文件，只有把它们的属性去除后，move命令才会执行。





## 4. find和findStr

1. 字符串查找：find
2. 字符串查找增强：findstr



**find（操作文件）**

> 格式：FIND [/V] [/C] [/N] [/I] [/OFF[LINE]] "string" \[[drive:]\[path]filename[ ...]]
>
> 基础写法：find "要查找的字符串" ”要查找的文件“
>
> ```
> find "国家有力量" 新时代的中国.txt			//打印出存在查找内容的整行文本
> ```
>
> 筛选指令：
>
> /v			打印出不存在"查找内容"的其他行文本
>
> /c			打印出有几行存在“查找内容”的数量
>

##  5. 常用特殊符号

3. &，&& ，|| 组合命令

5. % 变量引导符

**1. 重定向符（> 和 >>）**

> 功能：重定向符，将运行的结果传递给后面的文件或系统控制台。
>
> \>和\>>的唯一区别在于，\>是覆盖，\>>则是追加到最后。

```
1.txt
world
```

```
echo hello > 1.txt				//覆盖了world 结果 hello
```

```
echo world >> 1.txt				// hello
							   // world
```



**2. 命令管道符|**

> 功能：表示把在它之前的命令或语句的执行结果作为在它之后的命令或语句的处理对象

```
echo Y|rd /s c:\abc
```

管道命令|将echo y 的结果传给rd /s c:\abc 命令，从而达到自动输入y 的目的。



**3. 组合命令 &，&&，||**

> 功能：组合命令顾名思义就是把多个命令组合成一个命令使用。
>
> &：表并列，并列多少就执行多少，不看对错
>
> &&：表并列，直到执行到出现false的语句就停止，并返回false的结果
>
> ||：表并列，直到执行到出现true的语句出现就停止，并返回true的结果

1. &

   > 格式：第一条命令 & 第二条命令 [& 第三条命令...]

   ```
   dir z:\ & dir y:\ & dir c:\
   ```

   以上命令会连续显示z,y,c 盘的内容，不理会该盘是否存在。



2. && 

   > 格式：第一条命令 && 第二条命令 [&& 第三条命令...]

   ```
   dir z:\ && dir y:\ && dir c:\
   ```

   这个命令和上面的类似，但区别是遇到执行失败的命令后，后边的命令就不会再执行了。

3. ||

   > 格式：格式：第一条命令 || 第二条命令 [|| 第三条命令...]

   ```
   tasklist | find /i "qq.exe" && taskkill /f /im qq.exe || echo 你开了QQ?我不信
   ```

   通过&&和||两个符号可以认为把上例分成了三个命令：

   1. 第一个命令：tasklist | find /i "qq.exe"
   2. 第二个命令：tasklist /f  /im qq.exe
   3. 第三个命令：echo 你开了QQ?我不信

   这里我们分析一下：

   如果找不到QQ进程，第一个命令失败，则 && 不会执行第二个命令。命令不执行，则默认第二个也失败，则||会执行第三个命令：echo命令。

   如果找到了QQ进程，第一个命令成果，&&会继续执行第二个命令，taskkill命令，由于第二个命令执行成功，
   || 将不会执行第三个命令：echo 命令。

   



注意：优先级：管道命令（|） > 重定向（> 和 \>>）  > 组合命令（&，&& ,||）

```
dir c:\ && dir d:\ > a.txt
```

通常我们会觉得上面的语句会先执行前面的再执行后面的，但重定向的优先级> 高于 组合命令&&，因此只会先执行后边的

如果想要按顺序执行，可以在前面也加个重定向符

```
dir c:\ > a.txt && dir d:\ > a.txt
```

## 6. if条件语句

**if的5种条件语句**

1. 判断两个字符串是否相等。`if "字符串1" == "字符串2" command 语句`
2. 判断两个数值是否相等。`if  数值1 equ 数值2  command语句`
3. 判断磁盘，文件，文件夹是否存在。`if exist filename command语句`
4. 判断变量是否已定义，`if defined 变量 command语句`
5. 判断上个命令的返回值，`if errorlevel 数值 command语句`

**1. 判断两个字符串是否相等**

> `if "字符串1" == "字符串2" command 语句` 

```
if "a" == "A" (echo 输入的两个字符相同) else echo 输入的两个字符不同
```

如果要忽略大小写，请加 /i

```
if /i "a" == "A" (echo 输入的两个字符相同) else echo 输入的两个字符不同
```



**2. 判断两个数值是否相等**

> if 数值1 equ 数值2 command

| 中文含义   | 关系符 | 英文解释              |
| ---------- | ------ | --------------------- |
| 等于       | equ    | equal                 |
| 大于       | gtr    | greater than          |
| 大于或等于 | geq    | greater that or equal |
| 小于       | lss    | less than             |
| 小于或等于 | leq    | less than or equal    |
| 不等于     | neq    | no equal              |

```
if 12 geq 4 (echo 前面的大) else echo 后面的大			// 前面的大
```

对于字符串的数值，会一位一位的进行比较，

```
if "12" geq "4" (echo 前面的大) else echo 后面的大		//后面的大
```

两个元素作比较的流程是：先比较两个元素的首位，如果首位相同，再比较第二位，如果第二位相同，再比较第三位...以此类推



**3. 判断磁盘，文件，文件夹是否存在**

> `if exist filename command语句`

1. 判断盘符是否存在

```
if exist c: (echo C盘存在哦) else echo C盘不存在哦
```



2. 判断文件是否存在

```
if eixst 新时代的中国.txt (echo 文件存在哦) else 文件不存在哦
```

但这样还不够准确，因为文件夹也有可能取这样的名字

```
@echo off
dir /a-d 新时代的中国.txt >nul 2>nul
if %errorlevel%==0 (echo 新时代的中国.txt文件存在！) else echo 新时代的中国.txt文件不存在！
```

先用dir的/a-d参数去除 新时代的中国.txt  的目录属性，指定说明要搜索的 新时代的中国.txt 是文件而不是文件夹，并把结果（包括正确
和错误）屏蔽(>nul 2>nul)，如果dir找到了文件 新时代的中国.txt，那么其errorlevel值(dir命令的退出编码)会被设为0，否
则为1则是没有此文件。当然也可以用||和&&来判断。关于为什么要用if %errorlevel%==0 而不用if errorlevel 0
呢？在后面if errorlevel中会有说明！



3. 判断文件夹是否存在

```
if exist cpdd (echo cpdd文件夹存在) else echo cpdd文件夹不存在
```



**4. 判断变量是否已定义**

> `if defined 变量 command语句`
>
> 注意：变量不存在或值为空时，变量则为未定义

```
if defined a (echo 变量 a 已被定义) else echo 变量a未被定义
```



当我们用if defined 变量 command 语句判断变量是否被定义时，请注意 变量 为不使用引导符号%的变量名，不能用写为%变量%，否则出错。

```
set num=5
if defined num (echo 变量num已被定义) else (echo 变量num未被定义)			//变量num已被定义
```

```
set num=5
if defined %num% (echo 变量num已被定义) else (echo 变量num未被定义)			//变量num未被定义
```

注意1：变量前后不能接空格，否则结果就会是定义





**5. 判断上个命令的反回值**

> `if errorlevel数值 command语句`
>
> 功能：这个语句是用于判断上一个命令执行的返回值errorlevel，我们还是先来看看例子

语句特点：

1. 当使用 if errorlevel 0 command 句式时，他的含义是：如果返回的错误码值大于或等于0 的时候，将执行cmmand
2. 当使用if %errorlevel%==0 command句式时，它的含义是： 如果返回的错误码值等于0 的时候，将执行cmmand操作。



一般语句的返回值只有0（成功）和失败（1），但像变量的话怎样都是成功，所以这种情况不能用第一种句式

```
set /p input=请输入任意一条命令：
if errorlevel 0 (echo %input% 命令执行成功) else (echo %input% 命令执行失败)
```

用第一种句式,不管你前面的命令，是否执行成功,它都会认为命令成功了。何解？

因此我们需要用第二种句式，只有真的返回0的时候才会执行command语句



一般上一条命令的执行结果返回的值只有两个，"成功"用0 表示 "失败"用 1 表示，实际上，errorlevel 返回值可
以在0~255 之间，

```
例如xcopy 默认的errorlevel 值就有5 个，分别表示5 种执行状态：

0 复制文件成功
1 未找到复制文件
2 用户通过CTRL C 终止了xcopy操作
4 出现了初始化错误 
5 出现了磁盘写入错误
```



补充1：if not 语句
对于if 语句前面的5种基本语法，都可以加上not参数，如

1. if not "字符串1"=="字符串2" command 语句;

2. if not数值1 equ 数值2 command 语句；

3. If not exist filename command 语句;

4. if not defined 变量 command 语句；

5. if not errorlevel 数值 command 语句。

   

补充2：if语法嵌套

第一步

```
if  %a% geq 50 ( ) else echo a小于50
```

第二步

```
if %a% geq 50 (
	if %a % geq 80 () else echo a小于80
) else echo a小于50
```

第三步

```
if %a% geq 50 (
	if %a% geq 80 (
		if &a& geq 100 () else echo a小于100 
	) else echo a小于80
) else echo a小于50
```

第四步

```
if %a% geq 50 (
	if %a% geq 80 (
		if &a& geq 100 (
			echo a大于100
		) else echo a小于100 
	) else echo a小于80
) else echo a小于50
```
